Inspection Groups

The existing design system had been at a proof-of-concept stage for several years. It was created quickly with a skeleton set of components. It required a lot of custom design and implementation work. The existing design system also did not explicitly support any level of accessiblity.

Objective: Streamline Implementation

For an old, large, enterprise, government suite of applications the most expensive part of adding a new customer is implementation. It is expensive for both the customer and us. Hours of training and services are devoted to proper application set-up. The easier and faster we make it, the better for both parties.

Problems to Solve:

The administrative functionality for inspection groups was one of the most difficult pieces of functionality to implement because ...

  1. Parent/Child relationships are unclear
  2. Functionality and data is obscured
  3. Information architecture is ambiguous
  4. UI is too dense to scan

My Approach:

Using whiteboard and paper models I mapped the user’s journey and visualized proper data relationships.

I tested and refined these models with stakeholders, subject matter experts, and a small group of users

I explored various components, layers, and page combinations to represent models. Accela maintains a design system built on PrimeNG components.

I applied design system components to create a wireframe design to test with users and developers

Ultimately, I delivered functional and visual acceptance criteria to developers.
